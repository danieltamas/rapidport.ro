Workaround: temporarily stripped `framework` key before running CLI, restored it immediately after. The `framework` key was not in the final diff (restored to original value).

**Import aliases:** Generated files use `~/lib/utils` (Nuxt-compatible `~` alias). No `@/` imports found. The `shadcn-vue@latest` CLI with `"aliases": { "utils": "~/lib/utils" }` in `components.json` handled this automatically.

**New direct dependency added by CLI:** `@vueuse/core@14.2.1` — was already an indirect dep via other packages; CLI promoted it to direct dep in `package.json`. Two components require it: `Input.vue` (`useVModel`) and `TableEmpty.vue` (`reactiveOmit`).

**Extra file:** CLI generated `TableEmpty.vue` (not listed in spec's expected files) — this is a legitimate shadcn-vue addition, not a problem.
