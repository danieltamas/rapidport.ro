## Description

Create the minimal Nuxt 3 app scaffold that subsequent bootstrap tasks layer onto. This task ONLY installs Nuxt core and creates the empty shell. Theme, fonts, Mantine, Drizzle, env validation, primitives — each is its own separate task that follows this one.

Keep package.json churn small. Each bootstrap task adds only what it needs. Parallel parallelism comes AFTER bootstrap, not within it.

## Why It Matters

This is the foundation file every Phase 2 task builds on. Getting it right means the rest of Phase 2 doesn't fight framework config. Getting it wrong (e.g., loose TypeScript, wrong Nitro config, Nuxt 2 patterns) means weeks of friction.

## Acceptance Criteria

### Files created

- [ ] `app/package.json`
  - `"name": "rapidport-app"`
  - `"private": true`
  - `"type": "module"`
  - `"engines": { "node": ">=22" }`
  - scripts: `dev`, `build`, `generate`, `preview`, `postinstall` (runs `nuxi prepare`), `typecheck`, `lint` (placeholder — ESLint task is separate)
  - runtime deps: `nuxt@^3.13`, `vue@^3.5`, `zod@^3.23`
  - dev deps: `typescript@^5.5`, `@types/node@^22`, `vitest@^2`, `@vitest/coverage-v8@^2`
  - **no other deps in this task** — each bootstrap task adds its own
- [ ] `app/nuxt.config.ts` with:
  - `ssr: true`
  - `typescript: { strict: true, typeCheck: false }` — typecheck runs explicitly via `npm run typecheck` (which calls `nuxi typecheck`) and is enforced by CI + the `task-complete-gate.sh` hook. Setting `typeCheck: true` would pull `vue-tsc` as a peer dep and run it on every dev/build, slowing iteration. If we want in-build typecheck later, add `vue-tsc` to devDeps in a separate task.
  - `devtools: { enabled: false }` (avoid shipping devtools UI)
  - `devServer: { port: 3015 }` — **Rapidport dev port is 3015**, not Nuxt's default 3000 (3000 collides with Dani's other local services)
  - `nitro: { experimental: { websocket: true } }` (needed for SSE progress in Phase 2 api-jobs-events-sse)
  - `compatibilityDate: '2025-01-01'` (or latest Nuxt-recommended)
- [ ] `app/tsconfig.json`:
  - `"extends": "./.nuxt/tsconfig.json"` (Nuxt auto-generates the base)
  - `"compilerOptions": { "strict": true, "noUncheckedIndexedAccess": true, "noImplicitOverride": true }`
- [ ] `app/app.vue` — minimal shell: `<template><NuxtLayout><NuxtPage /></NuxtLayout></template>`
- [ ] `app/pages/index.vue` — placeholder: `<template><div>Rapidport — in progress</div></template>` (will be replaced in the `pages-landing` task; English is fine for placeholder since it's pre-launch)
- [ ] `app/.nvmrc` — content: `22`
- [ ] `app/.gitignore` (local, complements root `.gitignore`) — optional; can rely on root

### Verification commands (must all pass)

- [ ] `cd app && npm install` completes without errors (produces `app/package-lock.json`; commit it)
- [ ] `cd app && npx nuxi prepare` completes without errors (generates `.nuxt/`)
- [ ] `cd app && npx nuxi typecheck` passes
- [ ] `cd app && npm run dev` starts on **port 3015**, serves placeholder at `http://localhost:3015/`, no console errors (test: kill after verifying)
- [ ] `cd app && npm run build` produces `app/.output/` without errors

## Notes

- **Do NOT install all Phase 2 deps up front.** Each bootstrap task adds its own to keep PRs reviewable and package.json diffs legible.
- `app/node_modules/` is gitignored by the root `.gitignore`. Do NOT commit it.
- `app/.nuxt/` (generated) is gitignored. Do NOT commit it.
- `package-lock.json` MUST be committed — reproducible installs are a security property, not a nicety.
- If `npx nuxi typecheck` reports errors from generated files before any real code exists, run `npx nuxi prepare` first — Nuxt generates its type scaffolding on first run.
- Do NOT add any business logic, any theme, any server handlers, any DB code. Strictly scaffolding only. If you feel tempted, STOP — those belong in their own tasks.
- If you discover the Nuxt version has security advisories or the listed versions are incompatible, surface the finding in the DONE report's "Notes" section; do NOT silently bump to "latest".
- `nitro.experimental.websocket: true` enables WebSocket handlers later — it's forward-looking. If Nuxt's stable API changes this key, adapt.
